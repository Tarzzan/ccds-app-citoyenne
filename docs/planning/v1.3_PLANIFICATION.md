## CCDS Citoyen v1.3 — Document de Planification

> **Objectif :** Améliorer la qualité, l'accessibilité et l'engagement. La v1.3 se concentre sur la robustesse technique, l'inclusivité et l'introduction de fonctionnalités engageantes pour les citoyens.

---

### Thème 1 : Qualité Technique et Performance (Priorité P0)

**Objectif :** Assurer la stabilité, la maintenabilité et la performance de l'application.

| Ticket | Titre | Description | Priorité |
|---|---|---|---|
| **TEST-01** | Couverture de tests backend | Atteindre 80% de couverture de tests pour le backend PHP. Mettre en place des tests d'intégration pour les contrôleurs et des tests unitaires pour les services critiques (Permissions, Security). | **P0 - Critique** |
| **PERF-01** | Optimisation des performances mobiles | Réduire le temps de démarrage de l'application. Optimiser le rendu des listes avec `FlatList` et `SectionList`. Mettre en cache les images et les données fréquemment consultées. | **P1 - Important** |
| **TECH-02** | Nettoyage du backend | Supprimer les anciens fichiers de l'API procédurale (`backend/api/*.php`) et s'assurer que tout le trafic passe par le nouveau routeur OO. | **P0 - Critique** |

### Thème 2 : Accessibilité et Internationalisation (Priorité P1)

**Objectif :** Rendre l'application utilisable par le plus grand nombre, y compris les personnes en situation de handicap et les non-francophones.

| Ticket | Titre | Description | Priorité |
|---|---|---|---|
| **A11Y-01** | Conformité WCAG 2.1 AA (Mobile) | Auditer et corriger l'application mobile pour respecter les standards d'accessibilité : contrastes de couleurs, labels pour les lecteurs d'écran, navigation au clavier. | **P1 - Important** |
| **A11Y-02** | Mode Sombre (Dark Mode) | Implémenter un mode sombre complet sur l'application mobile, synchronisé avec les préférences du système d'exploitation. | **P2 - Normal** |
| **I18N-02** | Support multi-langue (Créole) | Ajouter le support pour le créole guyanais. Créer un fichier `cr.json` et permettre à l'utilisateur de changer de langue dans les paramètres. | **P2 - Normal** |

### Thème 3 : Engagement Citoyen (Priorité P2)

**Objectif :** Augmenter l'interaction et la rétention des utilisateurs en rendant l'application plus dynamique et gratifiante.

| Ticket | Titre | Description | Priorité |
|---|---|---|---|
| **GAMIF-01** | Gamification : Badges et Points | Introduire un système de points (par signalement, vote, commentaire) et de badges (ex: "Explorateur", "Contributeur Top 10%"). Créer un écran "Mon Impact" pour visualiser ses contributions. | **P2 - Normal** |
| **RT-01** | Carte temps réel (WebSocket) | Mettre à jour la carte principale en temps réel lorsque de nouveaux signalements sont créés. Utiliser une connexion WebSocket pour pousser les mises à jour aux clients connectés. | **P2 - Normal** |

### Migration SQL v1.3

```sql
-- Table pour la gamification
CREATE TABLE user_gamification (
  user_id INT PRIMARY KEY,
  points INT NOT NULL DEFAULT 0,
  last_action_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

CREATE TABLE user_badges (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  badge_key VARCHAR(50) NOT NULL,
  awarded_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY (user_id, badge_key),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- Table pour le support multi-langue
ALTER TABLE users ADD COLUMN language VARCHAR(5) NOT NULL DEFAULT 'fr';
```
