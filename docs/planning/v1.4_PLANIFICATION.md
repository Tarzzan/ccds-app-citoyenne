# Planification v1.4 — Industrialisation, Admin & Engagement

**Date :** 2026-03-04

## Contexte

Après les versions 1.0 (fondations), 1.1 (fonctionnalités clés), 1.2 (refactorisation OO) et 1.3 (qualité/accessibilité), la v1.4 se concentre sur l'industrialisation du projet, l'enrichissement des outils pour les administrateurs, et l'amélioration de l'engagement citoyen.

## Analyse des lacunes post-v1.3

1.  **DevOps / Industrialisation** : Le projet n'a ni CI/CD, ni conteneurisation (Docker), ni gestionnaire de dépendances PHP (Composer), ni système de migration de base de données automatisé. Le déploiement et les tests sont entièrement manuels, ce qui est un risque majeur.
2.  **Fonctionnalités Admin** : La gestion des utilisateurs est rudimentaire. Il manque des outils clés comme la génération de rapports PDF, une recherche globale, et un historique d'activité détaillé par citoyen.
3.  **Fonctionnalités Mobiles** : L'upload est limité à une seule photo par signalement. Le système de commentaires est trop simple (pas d'édition/suppression). Il n'y a aucune fonctionnalité de partage.
4.  **Documentation API** : La spécification OpenAPI est incomplète et ne reflète pas tous les endpoints des v1.2 et v1.3.

## Tickets v1.4

### Thème 1 : Industrialisation & DevOps (Priorité P0 — Critique)

| Ticket | Titre | Description | Priorité |
|---|---|---|---|
| DEVOPS-01 | CI/CD avec GitHub Actions | Mettre en place un workflow qui exécute automatiquement les tests PHPUnit et un linter (PHP-CS-Fixer) à chaque push sur `main`. | **P0** |
| DEVOPS-02 | Conteneurisation Docker | Créer un `docker-compose.yml` avec des services pour Nginx, PHP-FPM et MySQL, permettant de lancer un environnement de développement complet avec une seule commande. | **P0** |
| DEVOPS-03 | Intégration de Composer | Introduire `composer.json` pour gérer les dépendances PHP (ex: Ratchet pour WebSocket, PHPUnit, etc.) et l'autoloading PSR-4. | **P0** |
| DEVOPS-04 | Système de Migrations BDD | Intégrer Phinx pour gérer les migrations de base de données de manière versionnée et reproductible, remplaçant les fichiers SQL manuels. | **P0** |

### Thème 2 : Expérience Administrateur (Priorité P1 — Important)

| Ticket | Titre | Description | Priorité |
|---|---|---|---|
| ADMIN-04 | Gestion des utilisateurs avancée | Refondre la page `admin/users.php` avec recherche, filtres (rôle, statut), pagination, et une vue détail par utilisateur (infos, signalements, activité). | P1 |
| ADMIN-05 | Génération de rapports PDF | Sur la page de détail d'un incident, ajouter un bouton "Exporter en PDF" qui génère un rapport complet (infos, photos, commentaires, historique). | P1 |

### Thème 3 : Engagement Citoyen (Priorité P1 — Important)

| Ticket | Titre | Description | Priorité |
|---|---|---|---|

## Ordre de développement recommandé

1.  **DEVOPS-01, 02, 03, 04** : Traiter tout le bloc DevOps en premier pour industrialiser le projet.
2.  **ADMIN-04 + API-01** : Développer en parallèle la nouvelle gestion des utilisateurs (backend + frontend admin).
3.  **UX-04** : Gérer l'upload multiple, qui a un impact sur le backend et le frontend mobile.
4.  **ADMIN-05** : Implémenter la génération PDF (ex: avec FPDF ou TCPDF via Composer).
5.  **UX-05, UX-06, ADMIN-06, DOCS-01** : Regrouper les fonctionnalités restantes qui sont plus indépendantes.
