# Planification v1.5 — CCDS Citoyen

**Date** : 4 mars 2026
**Auteur** : Manus AI

## 1. Contexte

Après 4 versions majeures, le projet CCDS Citoyen a atteint une maturité fonctionnelle significative. La v1.4 a notamment permis d'industrialiser les fondations techniques (CI/CD, Docker, Composer, Phinx). L'analyse post-v1.4 révèle deux axes stratégiques pour la v1.5 :

1.  **Fiabilisation et Qualité** : Combler la dette technique accumulée, notamment sur la couverture de tests et les migrations de base de données.
2.  **Engagement et Confiance** : Introduire des fonctionnalités qui renforcent la confiance des utilisateurs (2FA, modération) et leur engagement (tableau de bord personnel).

## 2. Thèmes et Tickets v1.5

La v1.5 est organisée en **3 thèmes** et **10 tickets**.

### Thème 1 — Qualité et Dette Technique (P0 — Critique)

Ce thème est prioritaire pour garantir la stabilité et la maintenabilité du projet à long terme.

| Ticket | Titre | Description | Effort estimé |
|---|---|---|---|
| **TEST-02** | Couverture de tests backend 100% | Écrire les tests unitaires et d'intégration manquants pour les 9 contrôleurs introduits en v1.2, v1.3 et v1.4 (AuthController, CategoryController, CommentController, etc.). | **Élevé** |
| **DEVOPS-05** | Migrations Phinx pour v1.1-v1.4 | Créer les fichiers de migration Phinx pour toutes les modifications de schéma depuis la v1.1 (tables `votes`, `photos`, `user_gamification`, etc.). | **Moyen** |
| **TECH-03** | Consolidation des services API mobiles | Fusionner `api.ts` et `api_additions.ts` en un seul service API unifié et refactorisé. | **Faible** |
| **DOCS-02** | Documentation des composants React Native avec Storybook | Mettre en place Storybook pour documenter et visualiser les composants de l'interface utilisateur. | **Moyen** |

### Thème 2 — Sécurité et Confiance (P1 — Important)

Ces fonctionnalités visent à renforcer la sécurité des comptes et la qualité des contenus.

| Ticket | Titre | Description | Effort estimé |
|---|---|---|---|
| **SEC-03** | Authentification à deux facteurs (2FA) | Permettre aux utilisateurs d'activer la 2FA par email ou application (TOTP) pour sécuriser leur compte. | **Élevé** |
| **ADMIN-07** | Modération des commentaires | Système de signalement des commentaires inappropriés, avec une file d'attente de modération pour les administrateurs. | **Moyen** |
| **ADMIN-08** | Logs d'audit administrateur | Enregistrer toutes les actions sensibles effectuées dans le back-office (changement de rôle, suppression, etc.) pour une traçabilité complète. | **Moyen** |

### Thème 3 — Expérience Citoyen (P2 — Normal)

Ces fonctionnalités améliorent l'expérience utilisateur en offrant plus de personnalisation et de suivi.

| Ticket | Titre | Description | Effort estimé |
|---|---|---|---|
| **UX-07** | Tableau de bord citoyen | Nouvel écran "Mon Tableau de Bord" avec statistiques personnelles : nombre de signalements, votes, commentaires, statut moyen, etc. | **Moyen** |
| **UX-08** | Onboarding des nouveaux utilisateurs | Série d'écrans de bienvenue pour présenter les fonctionnalités clés de l'application lors de la première connexion. | **Faible** |
| **UX-09** | Page de statut publique | Créer une page web `statut.ccds.fr` affichant la disponibilité des services (API, WebSocket, Base de données) en temps réel. | **Moyen** |

## 3. Ordre de développement recommandé

1.  **Bloc Qualité (P0)** : Traiter `TEST-02`, `DEVOPS-05`, `TECH-03` et `DOCS-02` en priorité absolue.
2.  **Bloc Sécurité (P1)** : Développer `SEC-03`, `ADMIN-07` et `ADMIN-08`.
3.  **Bloc UX (P2)** : Terminer avec `UX-07`, `UX-08` et `UX-09`.

## 4. Migrations SQL prévisionnelles

```sql
-- Pour SEC-03 (2FA)
ALTER TABLE users
  ADD COLUMN two_factor_secret VARCHAR(255) NULL,
  ADD COLUMN two_factor_recovery_codes TEXT NULL,
  ADD COLUMN two_factor_method ENUM('none', 'email', 'totp') NOT NULL DEFAULT 'none';

-- Pour ADMIN-07 (Modération)
ALTER TABLE comments
  ADD COLUMN is_flagged TINYINT(1) NOT NULL DEFAULT 0,
  ADD COLUMN flagged_by INT NULL,
  ADD COLUMN moderated_at DATETIME NULL,
  ADD COLUMN moderation_status ENUM('pending', 'approved', 'rejected') NULL;

-- Pour ADMIN-08 (Logs d'audit)
CREATE TABLE audit_logs (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  action VARCHAR(255) NOT NULL,
  target_type VARCHAR(50) NULL,
  target_id INT NULL,
  details TEXT NULL,
  ip_address VARCHAR(45) NOT NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```
